# 我的 SBTI 版本

一个基于原项目复刻并做了定制化增强的纯前端版本。

这个版本保留了原测试的题库、结果体系和整体玩法，在不引入后端的前提下补充了更适合直接上线使用的能力，比如浏览器本地缓存、结果长图导出，以及更清晰的自定义入口。

## 项目特点

- 纯静态前端，无需数据库、服务端接口或构建流程
- 保留原测试逻辑与结果图资源
- 首页品牌信息支持集中配置
- 结果页支持一键保存长图
- 浏览器会自动缓存答题进度和最终结果
- 适合部署到 GitHub Pages、Cloudflare Pages、Vercel 静态站等平台

## 当前增强功能

### 1. 本地缓存恢复

- 用户作答过程中会自动保存答题进度到浏览器 `localStorage`
- 用户完成测试后，最终结果也会被缓存
- 刷新页面、关闭浏览器后再次打开，仍可继续查看上次进度或结果

### 2. 一键保存结果长图

- 结果页新增“保存结果长图”按钮
- 点击后会导出当前结果页为 PNG 图片
- 默认文件名类似：`我的 SBTI 版本-DEAD.png`

说明：

- 长图导出依赖运行时加载的 `html2canvas`
- 如果用户完全离线，导出按钮可能无法正常工作
- 缓存恢复功能不依赖外部 CDN，离线仍可用

## 本地运行

这是一个纯静态项目，不需要安装依赖。

### 方式一：用 Python 启动本地服务

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://127.0.0.1:4173
```

### 方式二：直接双击打开

也可以直接打开 `index.html`，但为了保证图片、缓存和导出行为更稳定，仍然建议通过本地静态服务预览。

## 自定义入口

主要自定义都集中在 [`index.html`](./index.html)。

### 1. 修改站点品牌信息

编辑 `SITE_CONFIG`：

```js
const SITE_CONFIG = {
  documentTitle: '我的 SBTI 版本',
  introTitle: '这是我的 SBTI 版本。',
  versionOwner: '当前版本：你的自定义版',
  deployInfo: '托管：待替换为你的部署平台',
  domainInfo: '域名：待替换为你的域名'
};
```

可调整内容：

- 页面标题
- 首页主标题
- 当前版本标识
- 托管信息
- 域名信息

### 2. 修改题目内容

编辑 [`index.html`](./index.html) 中的：

- `questions`
- `specialQuestions`

### 3. 修改人格结果与文案

编辑 [`index.html`](./index.html) 中的：

- `TYPE_LIBRARY`
- `NORMAL_TYPES`
- `TYPE_IMAGES`
- `DIM_EXPLANATIONS`

### 4. 替换结果图

将对应图片替换到 [`image`](./image) 目录，并保持文件名与 `TYPE_IMAGES` 中的映射一致。

## 部署方式

本项目没有构建步骤，部署时直接发布仓库根目录即可。

### GitHub Pages

- 将仓库推送到 GitHub
- 在仓库设置中开启 Pages
- 发布源选择默认分支或对应目录

### Cloudflare Pages

- 导入当前 GitHub 仓库
- 构建命令留空
- 输出目录填写 `/`

### 其他静态托管平台

只要平台支持直接托管 HTML/CSS/JS/图片静态资源，通常都可以直接部署。

## 适用场景

适合：

- 个人娱乐项目
- 社交传播页
- 低成本静态站点部署
- 无后端活动页或测试页

不适合直接拿来做：

- 严肃心理评估
- 需要账号体系和云端存档的产品
- 强依赖服务端统计或风控的业务系统

## 说明与注意事项

- 当前缓存基于浏览器本地存储，不同浏览器或无痕模式之间不会同步
- 清除浏览器站点数据后，缓存结果会一并消失
- 结果页导出的长图为当前页面内容截图，依赖浏览器能力
- 如果后续要做 SEO、统计、分享卡片、域名接入，可以继续在这个版本上扩展
